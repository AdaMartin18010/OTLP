## 📋 归档说明

本目录用于归档以下类型的文档：

1. **历史完成报告** - 各阶段的项目完成报告
2. **历史总结报告** - 项目的各类总结报告
3. **临时报告** - 执行过程中的临时报告
4. **过时文档** - 已被新版本替代的文档

**归档原则**:

- 保留历史记录，但不在根目录显示
- 最新版本保留在根目录或相关目录
- 所有文档通过Git版本控制保留完整历史
- 定期审查，确保归档完整性

## 📝 维护说明

### 归档标准

以下文件类型应归档至此：

1. ✅ 阶段性完成报告（如"阶段X完成报告"）
2. ✅ 临时性清理报告（如"文档清理报告"）
3. ✅ 已解决的修复报告（如"结构修复报告"）
4. ✅ 历史版本的计划文档
5. ✅ 已被新版本替代的评价报告
6. ✅ 明确废弃的文档

### 不应归档的文件

以下文件类型应保留在根目录：

1. ❌ 项目入口文档（README.md, START_HERE.md）
2. ❌ 当前项目仪表板（PROJECT_DASHBOARD.md）
3. ❌ 最新版本的状态报告和评价报告
4. ❌ 部署运维指南（快速入门、Docker/K8s部署等）
5. ❌ 当前执行中的计划和路线图

### 审查周期

- **频率**: 建议每月审查一次
- **负责人**: OTLP项目团队
- **内容**: 检查是否有新的历史报告需要归档
