## 二、命名规范建议

### 规范格式

```
NN_类别_主题.md

其中:
- NN: 两位数字编号 (00-99)
- 类别: 文件内容分类 (中文或英文缩写)
- 主题: 文件主题描述
```

### 编号分配建议

| 编号范围 | 用途 |
|---------|------|
| 00-09 | 索引、概述、导航类 |
| 10-19 | 核心理论文档 |
| 20-29 | 技术实现文档 |
| 30-39 | 配置和部署文档 |
| 40-49 | 运维和故障排查 |
| 50-59 | 案例和实践 |
| 60-69 | 趋势和分析 |
| 70-79 | 评价和总结 |
| 80-89 | 指南和教程 |
| 90-99 | 版本、变更日志、参考 |

每篇文档应包含：

```yaml
---
title: 文档标题
description: 文档描述
version: OTLP v1.9.0
spec_version: v1.55.0
semconv_version: v1.40.0
date: 2026-03-17
category: 分类
tags: [标签1, 标签2]
status: published
---
```

标准结构应包含：

1. 标题和元信息
2. 摘要/概述
3. 目录
4. 正文内容（分章节）
5. 总结
6. 参考链接
