## v1.10.0 主要更新内容

### 1. Profiles信号正式发布 (GA)

```yaml
变更前 (v1.9.0):
  Profiles信号状态: Experimental/Beta
  稳定性: 不稳定，可能变更
  生产使用: 不推荐

变更后 (v1.10.0):
  Profiles信号状态: Stable/GA ⭐
  稳定性: 向后兼容保证
  生产使用: 推荐

影响:
  - 可以安心在生产环境使用Profiles
  - 与Trace/Metric/Log同等地位
  - 完整的多语言SDK支持
```

### 2. ExponentialHistogram稳定化

```yaml
变更前 (v1.9.0):
  状态: Experimental
  Zero Threshold: 不支持
  性能: 基准

变更后 (v1.10.0):
  状态: Stable ⭐
  Zero Threshold: 支持 (更好处理浮点精度)
  性能: 内存使用减少20-30%
```

### 3. Resource属性增强

```yaml
新增Resource属性 (v1.41.0):
  - cloud.resource_id: 云资源唯一标识
  - container.runtime: 容器运行时类型
  - device.id: 设备唯一标识
  - device.model.identifier: 设备型号标识
  - process.context_switch_type: 上下文切换类型
```

### 4. Collector配置改进

```yaml
配置增强 (v0.117.0):
  memory_limiter:
    - soft_limit_mib: 新增软限制
    - spike_limit_percentage: 百分比配置

  health_check:
    - path: 自定义健康检查路径
    - detailed: 详细健康状态

  pipelines:
    - retry: 自动重试配置
```
