## 2. 格式规范细则

### 2.1 Frontmatter (YAML头)

- 必须包含在文档开头
- 使用 `---` 分隔
- 必须字段:
  - `title`: 文档标题
  - `description`: 简短描述 (50字以内)
  - `version`: 适用版本
  - `date`: 创建/更新日期
  - `category`: 主分类
  - `tags`: 标签列表
  - `status`: 文档状态

### 2.2 标题层级

- 文档主标题: `# 标题` (一级)
- 主要章节: `## 章节` (二级)
- 子章节: `### 子章节` (三级)
- 小节: `#### 小节` (四级)

**禁止**:
- 跳级使用标题 (如直接用三级标题)
- 使用五级、六级标题

### 2.3 代码块

- 必须指定语言类型
- 使用围栏代码块 ```
- 支持语言: yaml, json, bash, go, python, java, javascript, rust

正确示例:
```yaml
key: value
```

错误示例:

```
key: value
```

### 2.4 表格

- 必须包含表头分隔行
- 对齐使用 `:` 标识

示例:

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
|:-------|:--------:|-------:|
| 值1    |   值2    |    值3 |
```

### 2.5 分隔线

- 章节之间使用 `---` 分隔
- 不要连续使用多条分隔线

### 2.6 强调和列表

- 重要内容使用 **粗体**
- 关键术语使用 *斜体*
- 无序列表使用 `-` 或 `*`
- 有序列表使用 `1.` `2.` `3.`

### 2.7 链接和图片

- 内部链接使用相对路径
- 外部链接使用完整URL
- 图片添加alt文本

## 3. 分类标准

文档必须归属以下分类之一：

1. **理论基础** - 数学基础、形式化方法、分布式理论
2. **标准规范** - OTLP协议、语义约定、数据模型
3. **核心实现** - SDK、Collector、采样性能
4. **部署运维** - Docker、K8s、监控、故障排查
5. **前沿技术** - GenAI、eBPF、Profiles、Arrow
6. **行业实战** - 电商、金融、制造等行业案例
7. **工具生态** - 开源/商业平台集成
8. **学术资源** - 论文、形式化证明、研究报告

## 4. 状态标识

- `draft` - 草稿，待完善
- `review` - 审核中，待确认
- `published` - 已发布，可使用
- `deprecated` - 已弃用，将删除

## 5. 标签规范

常用标签：

- 技术标签: `otlp`, `sdk`, `collector`, `traces`, `metrics`, `logs`
- 语言标签: `go`, `python`, `java`, `nodejs`, `rust`
- 特性标签: `genai`, `ebpf`, `profiles`, `arrow`, `wasm`
- 场景标签: `production`, `security`, `performance`, `cost-optimization`
