# OTLP 项目最终重构方案 v4.0

## 发现的核心问题

1. **根目录散落文件**: 11个文件仍在docs根目录
2. **目录重复**: 01理论基础/01理论基础_形式化框架、02标准规范/02标准规范_语义约定等
3. **文档命名混乱**: 数字前缀和中文前缀混用
4. **编号重复**: 同一编号多个目录（0、1、2、3、5、7、13）
5. **重复子目录**: 13_智能与IoT可观测性出现两次

## 最终解决方案

### 1. 合并重复目录

```
合并方案:
- 01_理论基础 + 01_理论基础_形式化框架 → 01_理论基础
- 02_标准规范 + 02_标准规范_语义约定 → 02_标准规范
- 03_核心实现 + 03_核心实现_组件架构 → 03_核心实现
- 07_工具生态 + 工具 → 07_工具生态
- 10_智能与IoT可观测性 + 13_智能与IoT可观测性 → 10_智能与IoT可观测性
```

### 2. 统一文档命名

全部采用数字编号前缀:
- `01_协议概述.md`
- `02_传输层_gRPC.md`
- ...

### 3. 清理根目录

所有文件移到对应子目录

### 4. 最终目录结构 (单一层级编号)

```
docs/
├── 00_参考资料/          (合并00版本跟踪+00入口导航+00速查手册)

```
