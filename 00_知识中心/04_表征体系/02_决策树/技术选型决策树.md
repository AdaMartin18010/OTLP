## 📋 决策分支2: 部署架构选择

### 问题: 选择什么部署模式?

```mermaid
flowchart TD
    A[选择部署模式] --> B{运行环境?}

    B -->|Kubernetes| C{集群规模?}
    B -->|虚拟机/裸机| D{机器数量?}
    B -->|Serverless| E{云平台?}
    B -->|混合云| F[混合部署模式]

    C -->|小规模<50节点| G[DaemonSet模式<br/>Collector per Node]
    C -->|大规模>50节点| H[Gateway模式<br/>集中式Collector]
    C -->|多租户| I[Sidecar模式<br/>应用隔离]

    D -->|单台| J[单机部署<br/>Agent模式]
    D -->|多台<20| K[Agent+Gateway<br/>分层部署]
    D -->|多台>20| L[分层部署<br/>Agent+Gateway+Backend]

    E -->|AWS Lambda| M[Lambda Layer<br/>无Collector]
    E -->|Azure Functions| N[Agent模式<br/>轻量Collector]
    E -->|Google Cloud Run| O[Sidecar模式<br/>Cloud Run Sidecar]

    F --> P[混合部署<br/>K8s+VM统一管理]
```

### 部署模式对比

| 模式 | 适用场景 | 复杂度 | 成本 | 性能 |
|:---|:---|:---:|:---:|:---:|
| **DaemonSet** | K8s集群 | 中 | 中 | 高 |
| **Gateway** | 大规模集群 | 高 | 中 | 高 |
| **Sidecar** | 多租户/隔离 | 高 | 高 | 中 |
| **Agent** | 单机/VM | 低 | 低 | 中 |
| **Serverless** | FaaS | 低 | 按需 | 高 |

## 📋 决策分支3: 采样策略选择

### 问题: 选择什么采样策略?

```mermaid
flowchart TD
    A[选择采样策略] --> B{流量规模?}

    B -->|小规模<100TPS| C[AlwaysOn<br/>全量采集]
    B -->|中规模100-10000TPS| D{关键需求?}
    B -->|大规模>10000TPS| E{成本敏感?}

    D -->|追踪所有请求| F[AlwaysOn<br/>全量采集]
    D -->|性能优先| G[Probabilistic<br/>概率采样]
    D -->|关注异常| H[ParentBased<br/>智能传递]

    E -->|是| I[TailBased<br/>尾部采样]
    E -->|否| J[RateLimiting<br/>限速采样]

    I --> K[保留异常Trace<br/>丢弃正常Trace]
    J --> L[固定速率采集<br/>成本可控]
```

### 采样策略速查

| 策略 | 适用场景 | 复杂度 | 效果 |
|:---|:---|:---:|:---:|
| **AlwaysOn** | 开发/测试/小规模 | 低 | 完整 |
| **AlwaysOff** | 关闭采集 | 低 | 无 |
| **Probabilistic** | 均匀采样 | 低 | 统计有效 |
| **RateLimiting** | 固定速率 | 中 | 成本可控 |
| **ParentBased** | 继承决策 | 中 | 追踪完整 |
| **TailBased** | 异常保留 | 高 | 精准 |

详细对比请参考 [采样策略对比矩阵](../03_多维矩阵/采样策略对比矩阵.md)

## 📋 决策分支5: 插桩方式选择

### 问题: 选择什么插桩方式?

```mermaid
flowchart TD
    A[选择插桩方式] --> B{是否接受代码修改?}

    B -->|是| C{精确控制需求?}
    B -->|否| D{运行环境?}

    C -->|高| E[手动SDK埋点<br/>完全控制]
    C -->|中| F[OpenTelemetry Operator<br/>自动注入]

    D -->|K8s| G{是否支持Auto?}
    D -->|VM/裸机| H{eBPF支持?}

    G -->|是| I[OpenTelemetry Operator<br/>⭐推荐]
    G -->|否| J[手动SDK埋点]

    H -->|是| K[eBPF自动插桩<br/>OBI/Beyla]
    H -->|否| L[手动SDK埋点]
```

### 插桩方式对比

| 方式 | 代码侵入 | 精确度 | 适用场景 |
|:---|:---:|:---:|:---|
| **手动SDK** | 高 | ⭐⭐⭐⭐⭐ | 精确控制、复杂场景 |
| **Auto-instrumentation** | 无 | ⭐⭐⭐⭐ | 快速启动、标准框架 |
| **Operator自动注入** | 无 | ⭐⭐⭐⭐ | K8s环境、零侵入 |
| **eBPF** | 无 | ⭐⭐⭐ | 内核级、性能敏感 |

详细对比请参考 [插桩方式选择决策树](./插桩方式选择决策树.md)

## 🎯 综合决策流程

### 完整技术栈选型流程

```mermaid
flowchart TD
    Start([开始]) --> Q1{语言栈?}

    Q1 -->|Java/Go/Python/Node/.NET| Lang[选择对应SDK]
    Q1 -->|其他| LangCheck{有SDK?}
    LangCheck -->|有| Lang
    LangCheck -->|无| Manual[手动实现/等待支持]

    Lang --> Q2{环境?}
    Q2 -->|K8s| DeployK8s{规模?}
    Q2 -->|VM| DeployVM{数量?}
    Q2 -->|Serverless| DeployServerless

    DeployK8s -->|小| K8sSmall[DaemonSet]
    DeployK8s -->|大| K8sLarge[Gateway模式]

    DeployVM -->|少| VMFew[Agent模式]
    DeployVM -->|多| VMMany[Agent+Gateway]

    DeployServerless --> Lambda[Lambda Layer]
    DeployServerless --> Function[Agent模式]

    K8sSmall --> Q3{流量?}
    K8sLarge --> Q3
    VMFew --> Q3
    VMMany --> Q3
    Lambda --> Q4{成本敏感?}
    Function --> Q4

    Q3 -->|小| Sample1[AlwaysOn]
    Q3 -->|中| Sample2[Probabilistic]
    Q3 -->|大| Sample3[TailBased]

    Sample1 --> Q4
    Sample2 --> Q4
    Sample3 --> Q4

    Q4 -->|是| Backend1[开源方案]
    Q4 -->|否| Backend2[商业方案]

    Backend1 --> Q5{规模?}
    Backend2 --> Q6{功能?}

    Q5 -->|小| BackendChoice1[SigNoz]
    Q5 -->|中| BackendChoice2[Tempo+VM]
    Q5 -->|大| BackendChoice3[ClickHouse]

    Q6 -->|全托管| BackendChoice4[Grafana Cloud]
    Q6 -->|企业级| BackendChoice5[Datadog]
    Q6 -->|平衡| BackendChoice6[SigNoz Cloud]

    BackendChoice1 --> Q7{插桩?}
    BackendChoice2 --> Q7
    BackendChoice3 --> Q7
    BackendChoice4 --> Q7
    BackendChoice5 --> Q7
    BackendChoice6 --> Q7

    Q7 -->|接受修改| Instrument1[手动SDK]
    Q7 -->|零侵入| Instrument2[Auto/Operator]
    Q7 -->|K8s| Instrument3[Operator]

    Instrument1 --> End([完成])
    Instrument2 --> End
    Instrument3 --> End
```

## 📊 决策检查清单

### 选型前必查项

```yaml
语言选择:
  - [ ] 团队熟悉度评估
  - [ ] SDK成熟度检查
  - [ ] 生态覆盖度确认
  - [ ] 性能要求匹配

部署选择:
  - [ ] 环境类型确认 (K8s/VM/Serverless)
  - [ ] 规模评估 (节点数/流量)
  - [ ] 网络拓扑设计
  - [ ] 高可用需求确认

采样选择:
  - [ ] 流量规模估算
  - [ ] 成本预算确认
  - [ ] 关键Trace保留需求
  - [ ] 合规要求检查

后端选择:
  - [ ] 数据量预估 (GB/天)
  - [ ] 预算限制确认
  - [ ] 功能需求清单
  - [ ] 扩展性要求

插桩选择:
  - [ ] 代码修改接受度
  - [ ] 精确控制需求
  - [ ] 快速启动需求
  - [ ] 性能敏感程度
```
