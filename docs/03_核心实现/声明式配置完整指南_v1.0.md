## 什么是声明式配置?

### 核心概念

**声明式配置**是OpenTelemetry标准化的配置方式，使用YAML文件定义SDK和Collector的完整配置，取代传统的代码配置或环境变量配置。

```
┌─────────────────────────────────────────────────────────────┐
│                    配置方式演进                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  方式1: 代码配置 (传统)                                      │
│    优点: 灵活、类型安全                                       │
│    缺点: 需要重新编译、耦合代码                                │
│                                                             │
│  方式2: 环境变量 (过渡)                                      │
│    优点: 运行时配置                                          │
│    缺点: 能力有限、难以管理复杂配置                              │
│                                                             │
│  方式3: 声明式配置 (推荐) v1.0.0                              │
│    优点: 标准化、验证、自动补全、跨语言一致                      │
│    缺点: 需要学习新格式                                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## � 架构设计

### 三层配置模型

```yaml
# Level 1: 全局配置
file_format: "0.3"           # 文件格式版本

# Level 2: 提供者配置
provider:
  name: "my-service"
  version: "1.0.0"

# Level 3: 信号配置
tracer_provider:             # 追踪配置
meter_provider:              # 指标配置
logger_provider:             # 日志配置
```

## 完整配置示例

### 基础配置

```yaml
# config.yaml
file_format: "0.3"

# 资源属性
resource:
  attributes:
    - name: service.name
      value: "my-application"
    - name: service.version
      value: "1.0.0"
    - name: deployment.environment
      value: "production"

# 追踪配置
tracer_provider:
  processors:
    - batch:
        exporter:
          otlp:
            endpoint: http://localhost:4317
            protocol: grpc
        schedule_delay: 5000
        max_queue_size: 2048
        max_export_batch_size: 512

  sampler:
    parent_based:
      root:
        trace_id_ratio_based:
          ratio: 1.0

# 指标配置
meter_provider:
  readers:
    - periodic:
        exporter:
          otlp:
            endpoint: http://localhost:4317
            protocol: grpc
        interval: 60000

# 日志配置
logger_provider:
  processors:
    - batch:
        exporter:
          otlp:
            endpoint: http://localhost:4317
            protocol: grpc

# 属性限制
attribute_limits:
  attribute_value_length_limit: 4096
  attribute_count_limit: 128
```

### 高级配置

#### TLS配置 (v1.0新标准)

```yaml
# v1.0 TLS配置格式
tracer_provider:
  processors:
    - batch:
        exporter:
          otlp_http:
            endpoint: https://otel-collector:4318
            tls:
              ca_file: /etc/ssl/certs/ca.crt
              cert_file: /etc/ssl/certs/client.crt
              key_file: /etc/ssl/private/client.key
              # 或使用内联证书
              # ca_pem: |
              #   -----BEGIN CERTIFICATE-----
              #   ...
              #   -----END CERTIFICATE-----
```

#### 多导出器配置

```yaml
# 多导出器 - traces分发到多个后端
tracer_provider:
  processors:
    - batch:
        exporters:
          - otlp:
              endpoint: http://jaeger:4317
              protocol: grpc
          - otlp:
              endpoint: http://datadog:4317
              protocol: grpc
          - console: {}  # 调试输出
```

#### 采样策略配置

```yaml
# 高级采样配置
sampler:
  parent_based:
    root:
      rule_based:
        rules:
          - name: "error-sampling"
            condition:
              attribute_equals:
                key: http.status_code
                value: "500"
            sampler:
              always_on: {}
          - name: "slow-request-sampling"
            condition:
              attribute_greater_than:
                key: http.response.duration_ms
                value: 1000
            sampler:
              always_on: {}
          - name: "default"
            sampler:
              trace_id_ratio_based:
                ratio: 0.1
```

## 各语言SDK配置

### Go SDK

```go
package main

import (
    "context"
    "log"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/sdk/resource"
    "go.opentelemetry.io/otel/sdk/trace"
)

func main() {
    // 从文件加载配置
    config, err := loadConfig("config.yaml")
    if err != nil {
        log.Fatal(err)
    }

    // 应用配置
    sdk, err := config.TracerProvider()
    if err != nil {
        log.Fatal(err)
    }

    // 设置为全局provider
    otel.SetTracerProvider(sdk)

    defer sdk.Shutdown(context.Background())

    // 使用...
}
```

### Python SDK

```python
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.config import load_config

# 加载配置
config = load_config("config.yaml")

# 创建provider
provider = TracerProvider(config=config.tracer_provider)
trace.set_tracer_provider(provider)

# 使用
tracer = trace.get_tracer(__name__)
```

### Java SDK

```java
import io.opentelemetry.sdk.OpenTelemetrySdk;
import io.opentelemetry.config.DeclarativeConfig;

public class Main {
    public static void main(String[] args) {
        // 加载配置
        DeclarativeConfig config = DeclarativeConfig.load("config.yaml");

        // 创建SDK
        OpenTelemetrySdk sdk = config.createSdk();

        // 使用...
    }
}
```

### Node.js SDK

```javascript
const { NodeSDK } = require('@opentelemetry/sdk-node');
const { loadConfig } = require('@opentelemetry/config');

// 加载配置
const config = loadConfig('config.yaml');

// 创建SDK
const sdk = new NodeSDK(config);

// 启动
sdk.start();

// 关闭
process.on('SIGTERM', () => {
  sdk.shutdown()
    .then(() => console.log('SDK shut down'))
    .catch((err) => console.error('Error shutting down SDK', err));
});
```

## 与传统配置对比

### 环境变量配置 (旧方式)

```bash
export OTEL_SERVICE_NAME="my-service"
export OTEL_SERVICE_VERSION="1.0.0"
export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4317"
export OTEL_TRACES_SAMPLER="parentbased_traceidratio"
export OTEL_TRACES_SAMPLER_ARG="0.1"
export OTEL_RESOURCE_ATTRIBUTES="deployment.environment=production,host.name=server-01"
```

### 声明式配置 (新方式)

```yaml
file_format: "0.3"

resource:
  attributes:
    - name: service.name
      value: "my-service"
    - name: service.version
      value: "1.0.0"
    - name: deployment.environment
      value: "production"
    - name: host.name
      value: "server-01"

tracer_provider:
  sampler:
    parent_based:
      root:
        trace_id_ratio_based:
          ratio: 0.1
  processors:
    - batch:
        exporter:
          otlp:
            endpoint: http://localhost:4317
```

## 生产环境最佳实践

### 配置分离

```yaml
# base.yaml - 基础配置
file_format: "0.3"

resource:
  attributes:
    - name: service.name
      value: "my-service"

---
# production.yaml - 生产环境覆盖
file_format: "0.3"

resource:
  attributes:
    - name: deployment.environment
      value: "production"

tracer_provider:
  sampler:
    parent_based:
      root:
        trace_id_ratio_based:
          ratio: 0.1  # 生产环境采样率
```

### 敏感信息处理

```yaml
# 使用环境变量引用
file_format: "0.3"

tracer_provider:
  processors:
    - batch:
        exporter:
          otlp:
            endpoint: ${OTEL_EXPORTER_ENDPOINT}
            headers:
              - name: api-key
                value: ${OTEL_API_KEY}  # 从环境变量读取
```
