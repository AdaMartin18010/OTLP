# 自适应采样算法详解

## 1. 采样基础理论

### 1.1 为什么需要采样

- 数据量爆炸: 100K QPS × 100 spans = 10M spans/秒
- 存储成本高昂: 每天数TB存储
- 查询性能下降
- 网络带宽占用

### 1.2 采样误差分析

| 采样率 | 相对误差 | 适用场景 |
|--------|----------|----------|
| 100% | 0% | 关键路径 |
| 10% | ±3.2% | 普通业务 |
| 1% | ±10% | 高流量服务 |
| 0.1% | ±31.6% | 日志型数据 |

## 2. 概率采样

### 2.1 基于Trace ID的采样

```go
type TraceIDRatioBased struct {
    ratio    float64
    boundary uint64
}

func (s *TraceIDRatioBased) ShouldSample(traceID string) bool {
    traceIDInt := binary.BigEndian.Uint64(traceIDBytes[:8])
    return traceIDInt < s.boundary
}
```

## 3. 尾部采样

### 3.1 Collector尾部采样配置

```yaml
processors:
  tail_sampling:
    decision_wait: 10s
    num_traces: 100000
    policies:
      - name: errors
        type: status_code
        status_code: {status_codes: [ERROR]}
      - name: slow_requests
        type: latency
        latency: {threshold_ms: 1000}
```
