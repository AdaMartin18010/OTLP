# 微服务可观测性最佳实践

> **适用架构**: Kubernetes + 微服务
> **技术深度**: ⭐⭐⭐⭐ (高级)

## 1. 微服务可观测性挑战

### 1.1 分布式系统复杂性

```
微服务可观测性挑战:
├── 请求路径长
│   └── 单个请求可能经过10+服务
├── 故障传播快
│   └── 级联故障难以定位
├── 依赖关系复杂
│   └── 服务网状依赖
├── 环境动态变化
│   └── Pod生命周期短
└── 数据量巨大
    └── 1000+服务的观测数据
```

### 1.2 关键观测维度

```
微服务观测三大支柱:

分布式追踪 (Traces)
├── 请求链路可视化
├── 延迟瓶颈定位
├── 服务依赖映射
└── 故障影响分析

指标监控 (Metrics)
├── 服务QPS/延迟
├── 资源使用率
├── 错误率监控
└── 容量规划

日志聚合 (Logs)
├── 结构化日志
├── 上下文关联
├── 错误分析
└── 审计追踪
```

## 2. 服务网格集成

### 2.1 Istio + OpenTelemetry

```yaml
# Istio Telemetry配置
apiVersion: telemetry.istio.io/v1alpha1
kind: Telemetry
metadata:
  name: otel-metrics
  namespace: istio-system
spec:
  metrics:
    - providers:
        - name: prometheus
      overrides:
        - match:
            metric: REQUEST_COUNT
          tagOverrides:
            destination_service:
              operation: UPSERT
              value: "unknown"
  accessLogging:
    - providers:
        - name: otel-collector
      filter:
        expression: "response.code >= 400"
```

### 2.2 Sidecar自动注入

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: payment-service
spec:
  template:
    metadata:
      annotations:
        # 启用OTel Sidecar注入
        instrumentation.opentelemetry.io/inject-sdk: "true"
        # Istio Sidecar配置
        proxy.istio.io/config: |
          tracing:
            sampling: 10.0
            custom_tags:
              environment:
                literal:
                  value: production
    spec:
      containers:
        - name: app
          image: payment-service:v1.0.0
```

## 3. 分布式追踪最佳实践

### 3.1 上下文传播

```java
// Java: 跨服务上下文传播
@RestController
public class OrderController {

    @Autowired
    private RestTemplate restTemplate;

    @PostMapping("/orders")
    public Order createOrder(@RequestBody OrderRequest request) {
        // 当前span自动传播到下游服务
        ResponseEntity<Payment> payment = restTemplate.postForEntity(
            "http://payment-service/payments",
            request.getPayment(),
            Payment.class
        );

        // 库存扣减
        restTemplate.postForObject(
            "http://inventory-service/reserve",
            request.getItems(),
            Void.class
        );

        return orderService.create(request);
    }
}
```

### 3.2 自定义Span

```java
// 业务操作追踪
@Service
public class PaymentService {

    private final Tracer tracer;

    public Payment processPayment(PaymentRequest request) {
        // 创建自定义span
        Span span = tracer.spanBuilder("payment.process")
            .setAttribute("payment.method", request.getMethod())
            .setAttribute("payment.amount", request.getAmount())
            .startSpan();

        try (Scope scope = span.makeCurrent()) {
            // 验证
            validatePayment(request);
            span.addEvent("validation_complete");

            // 扣款
            charge(request);
            span.addEvent("charge_complete");

            // 记录成功
            span.setStatus(StatusCode.OK);
            return payment;

        } catch (PaymentException e) {
            span.recordException(e);
            span.setStatus(StatusCode.ERROR, e.getMessage());
            throw e;
        } finally {
            span.end();
        }
    }
}
```

## 4. 服务依赖分析

### 4.1 依赖拓扑

```promql
# 服务调用拓扑
sum(rate(http_client_duration_count[5m])) by (service, peer_service)

# 依赖延迟
histogram_quantile(0.95,
  sum(rate(http_client_duration_bucket[5m])) by (le, peer_service)
)

# 错误依赖
sum(rate(http_client_errors[5m])) by (peer_service)
```
