## 一、可视化设计原则

### 1.1 信息层级架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     可观测性信息金字塔                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                              ┌─────────┐                        │
│                              │  总览   │  ← 30秒获取系统状态    │
│                              │ Status  │                        │
│                              └────┬────┘                        │
│                                   │                             │
│                         ┌─────────┴─────────┐                   │
│                         │       概览         │  ← 2分钟定位问题  │
│                         │    Overview       │                   │
│                         └─────────┬─────────┘                   │
│                                   │                             │
│              ┌────────────────────┼────────────────────┐        │
│              │                    │                    │        │
│        ┌─────┴─────┐        ┌─────┴─────┐        ┌─────┴─────┐  │
│        │   详细    │        │   详细    │        │   详细    │  │
│        │  Detail   │        │  Detail   │        │  Detail   │  │
│        └───────────┘        └───────────┘        └───────────┘  │
│                                                                 │
│  设计原则:                                                       │
│  • 每个层级回答不同问题                                          │
│  • 支持下钻(Drill-down)导航                                      │
│  • 避免信息过载                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则详解

```yaml
# 可观测性可视化设计原则
design_principles:

  认知负荷管理:
    - 原则: "5秒法则"
      描述: "用户应在5秒内理解当前系统状态"

    - 原则: "渐进披露"
      描述: "根据用户需求逐步展示详细信息"

  一致性:
    - 原则: "颜色语义一致性"
      标准:
        green: "正常/健康"
        yellow: "警告/注意"
        red: "严重/故障"
        blue: "信息/进行中"

  可访问性:
    - 原则: "色盲友好"
      实施: "同时使用颜色和图标，避免红绿对比"
```

### 2.2 Grafana Dashboard JSON配置

```json
{
  "dashboard": {
    "title": "服务可观测性 - 标准模板",
    "tags": ["observability", "service"],
    "refresh": "30s",

    "templating": {
      "list": [{
        "name": "service",
        "type": "query",
        "query": "label_values(http_requests_total, service)"
      }]
    },

    "panels": [
      {
        "id": 1,
        "title": "可用性",
        "type": "stat",
        "targets": [{
          "expr": "sum(rate(http_requests_total{service=\"$service\",status=~\"2..|3..\"}[5m])) / sum(rate(http_requests_total{service=\"$service\"}[5m]))"
        }],
        "fieldConfig": {
          "defaults": {
            "unit": "percentunit",
            "thresholds": {
              "steps": [
                {"color": "red", "value": 0},
                {"color": "yellow", "value": 0.99},
                {"color": "green", "value": 0.999}
              ]
            }
          }
        }
      },
      {
        "id": 2,
        "title": "P99延迟",
        "type": "stat",
        "targets": [{
          "expr": "histogram_quantile(0.99, sum(rate(http_request_duration_seconds_bucket{service=\"$service\"}[5m])) by (le))"
        }],
        "fieldConfig": {
          "defaults": {
            "unit": "s",
            "thresholds": {
              "steps": [
                {"color": "green", "value": 0},
                {"color": "yellow", "value": 0.1},
                {"color": "red", "value": 0.5}
              ]
            }
          }
        }
      },
      {
        "id": 3,
        "title": "延迟趋势",
        "type": "timeseries",
        "targets": [
          {
            "expr": "histogram_quantile(0.50, sum(rate(http_request_duration_seconds_bucket{service=\"$service\"}[5m])) by (le))",
            "legendFormat": "P50"
          },
          {
            "expr": "histogram_quantile(0.99, sum(rate(http_request_duration_seconds_bucket{service=\"$service\"}[5m])) by (le))",
            "legendFormat": "P99"
          }
        ]
      }
    ]
  }
}
```

### 4.1 Dashboard管理规范

```yaml
# Grafana Dashboard管理规范
dashboard_management:

  命名规范:
    format: "[团队]-[服务]-[环境]-[用途]"
    examples:
      - "[PAY]-PaymentAPI-prod-overview"
      - "[SRE]-Infrastructure-prod-detail"

  版本控制:
    - 使用Grafana Provisioning管理仪表板JSON
    - 将仪表板定义存储在Git仓库
    - 使用CI/CD自动部署

  权限管理:
    folders:
      - name: "Production"
        permissions: "viewer:All, editor:SRE"
```

### 4.2 性能优化

```yaml
# 可视化性能优化
performance_optimization:

  查询优化:
    - 使用录制规则预计算复杂查询
    - 限制查询时间范围

  渲染优化:
    - 限制每面板数据点数量(<1000)
    - 使用服务器端渲染

  资源管理:
    - 单个仪表板不超过20个面板
    - 复杂仪表板拆分为多个子仪表板
```
