## 📊 关键指标

### 本模块关注的核心指标

| 指标类别 | 指标名称 | 说明 | 目标值 |
|----------|----------|------|--------|
| **可靠性** | SLO达成率 | 实际服务达到SLO目标的比例 | >99% |
| **效率** | MTTR | 平均恢复时间 | <30分钟 |
| **效率** | 告警准确性 | 有效告警/总告警 | >80% |
| **成本** | 资源利用率 | CPU/内存平均利用率 | 40-70% |
| **成本** | 单位请求成本 | 每百万请求基础设施成本 | 持续优化 |
| **决策** | 数据驱动决策比例 | 基于数据的技术决策占比 | >80% |

## 🚀 快速开始

### 1. 环境准备

```bash
# 克隆OTLP示例代码
git clone https://github.com/otlp/examples.git

# 启动本地演示环境
docker-compose -f docker-compose.analytics.yml up -d

# 访问Grafana
open http://localhost:3000
```

### 2. 第一个SLO

```python
# 使用Sloth定义SLO
# my-service-slo.yaml

service: my-api
slos:
  - name: availability
    objective: 99.9
    sli:
      events:
        error_query: sum(rate(http_requests_total{status=~"5.."}[{{.window}}]))
        total_query: sum(rate(http_requests_total[{{.window}}]))
```

```bash
# 生成Prometheus规则
sloth generate -i my-service-slo.yaml -o prometheus-rules.yaml

# 应用规则
kubectl apply -f prometheus-rules.yaml
```
