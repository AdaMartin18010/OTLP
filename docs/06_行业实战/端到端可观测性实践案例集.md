## 案例1: 电商平台全链路可观测性

### 1.1 场景描述

**业务背景**: 某中型电商平台，日均订单10万+，微服务架构50+服务

**痛点**:

- 下单链路涉及12个服务，故障定位困难
- 大促期间系统瓶颈无法快速识别
- 用户体验问题缺乏量化数据

**目标**:

- 实现全链路追踪，故障定位时间从30分钟缩短到5分钟
- 建立性能基线，提前发现瓶颈
- 量化用户体验，支持产品优化

#### 步骤2: 服务接入

**Go服务接入**:

```go
// main.go
package main

import (
    "context"
    "log"

    "go.opentelemetry.io/otel"
    "go.opentelemetry.io/otel/exporters/otlp/otlptrace/otlptracegrpc"
    "go.opentelemetry.io/otel/sdk/resource"
    sdktrace "go.opentelemetry.io/otel/sdk/trace"
    semconv "go.opentelemetry.io/otel/semconv/v1.24.0"
)

func initTracer() *sdktrace.TracerProvider {
    ctx := context.Background()

    exporter, err := otlptracegrpc.New(ctx,
        otlptracegrpc.WithEndpoint("otel-collector:4317"),
        otlptracegrpc.WithInsecure(),
    )
    if err != nil {
        log.Fatal(err)
    }

    tp := sdktrace.NewTracerProvider(
        sdktrace.WithBatcher(exporter),
        sdktrace.WithResource(resource.NewWithAttributes(
            semconv.SchemaURL,
            semconv.ServiceNameKey.String("order-service"),
            semconv.ServiceVersionKey.String("1.0.0"),
        )),
    )

    otel.SetTracerProvider(tp)
    return tp
}
```

#### 步骤3: 业务埋点

**订单创建链路**:

```go
func (s *OrderService) CreateOrder(ctx context.Context, req *CreateOrderRequest) (*Order, error) {
    tracer := otel.Tracer("order-service")
    ctx, span := tracer.Start(ctx, "CreateOrder",
        trace.WithAttributes(
            attribute.String("user.id", req.UserID),
            attribute.String("product.id", req.ProductID),
            attribute.Int("quantity", req.Quantity),
        ),
    )
    defer span.End()

    // 1. 检查库存
    if err := s.checkInventory(ctx, req.ProductID, req.Quantity); err != nil {
        span.RecordError(err)
        span.SetStatus(codes.Error, "库存不足")
        return nil, err
    }

    // 2. 计算价格
    price, err := s.calculatePrice(ctx, req)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }
    span.SetAttributes(attribute.Float64("order.amount", price))

    // 3. 创建订单
    order, err := s.saveOrder(ctx, req, price)
    if err != nil {
        span.RecordError(err)
        return nil, err
    }

    span.SetAttributes(attribute.String("order.id", order.ID))
    span.SetStatus(codes.Ok, "订单创建成功")

    return order, nil
}
```

### 2.3 安全合规配置

```yaml
# collector-security.yaml
processors:
  # PII数据脱敏
  attributes/pii:
    actions:
      - key: user.ssn
        action: hash
      - key: user.phone
        action: mask
        pattern: "(\d{3})\d{4}(\d{4})"
        replacement: "$1****$2"
      - key: account.number
        action: delete

  # 审计日志
  resource/audit:
    attributes:
      - key: audit.timestamp
        from_attribute: timestamp
        action: upsert
      - key: audit.source
        value: "core-banking"
        action: upsert

exporters:
  # 加密传输
  otlp/secure:
    endpoint: backend:4317
    tls:
      cert_file: /certs/client.crt
      key_file: /certs/client.key
      ca_file: /certs/ca.crt
```

### 3.3 边缘Collector配置

```yaml
# edge-collector.yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

processors:
  # 本地聚合
  batch:
    timeout: 10s
    send_batch_size: 100

  # 内存限制
  memory_limiter:
    limit_mib: 100
    spike_limit_mib: 20

  # 智能采样
  tail_sampling:
    decision_wait: 10s
    num_traces: 100
    expected_new_traces_per_sec: 10
    policies:
      - name: errors
        type: status_code
        status_code: {status_codes: [ERROR]}
      - name: slow_requests
        type: latency
        latency: {threshold_ms: 1000}

exporters:
  # 本地缓存
  file:
    path: /data/otel-backup

  # 云端传输
  otlp/cloud:
    endpoint: cloud-collector:4317
    compression: gzip
    retry_on_failure:
      enabled: true
      initial_interval: 5s
      max_interval: 30s
      max_elapsed_time: 300s
    sending_queue:
      enabled: true
      num_consumers: 2
      queue_size: 1000

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [memory_limiter, tail_sampling, batch]
      exporters: [file, otlp/cloud]
```

### 常见场景解决方案

| 场景 | 解决方案 | 关键配置 |
|------|----------|----------|
| 高并发 | 批量导出+采样 | `batch.timeout=1s`, `TraceIDRatioBased(0.1)` |

| 遗留系统 | Sidecar模式 | Envoy自动采集 |
