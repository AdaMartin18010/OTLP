## 1.  传输安全

### 1.1 TLS配置 (必选)

**Collector接收端TLS**:
`yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
        tls:
          cert_file: /etc/otel/server.crt
          key_file: /etc/otel/server.key
          ca_file: /etc/otel/ca.crt  # 可选: 用于mTLS
          require_client_certificate: true  # 启用mTLS
      http:
        endpoint: 0.0.0.0:4318
        tls:
          cert_file: /etc/otel/server.crt
          key_file: /etc/otel/server.key
`

**证书要求**:

- [ ] 使用生产级证书 (非自签名)
- [ ] 证书有效期监控 (<30天告警)
- [ ] 私钥权限设置 (chmod 600)
- [ ] 定期轮换策略 (每90天)

### 1.2 TLS版本与加密套件

**最低要求**:

- TLS 1.2+ (TLS 1.3推荐)
- 禁用弱加密算法

**Nginx配置示例**:
`
ginx
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers 'ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256';
ssl_prefer_server_ciphers on;
`

## 2. � 认证与授权

### 2.1 Token认证

**Bearer Token配置**:
`yaml
extensions:
  bearertokenauth:
    scheme: Bearer
    tokens:
      - token: \

receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
        auth:
          authenticator: bearertokenauth
`

**Token管理**:

- [ ] Token定期轮换
- [ ] Token泄露响应流程
- [ ] Token最小权限原则
- [ ] Token存储安全 (Key Vault)

### 2.2 mTLS (双向认证)

**适用场景**: 高安全要求环境

**配置**:
`yaml
receivers:
  otlp:
    protocols:
      grpc:
        tls:
          cert_file: /etc/otel/server.crt
          key_file: /etc/otel/server.key
          ca_file: /etc/otel/ca.crt
          require_client_certificate: true
`

## 3. � 数据脱敏

### 3.1 敏感数据处理

**Processor配置**:
`yaml
processors:
  attributes/redact:
    actions:
      - key: password
        action: delete
      - key: token
        action: hash
      - key: credit_card
        action: mask
        mask_char: '*'
        mask_length: 4
      - key: email
        action: hash
      - key: phone
        action: mask
        pattern: \+(\d{2})\d*\d{4}
        replacement: +\****\

  resource/delete_sensitive:
    attributes:
      - key: process.command_line
        action: delete
      - key: process.executable.path
        action: delete
`

### 3.2 脱敏规则

| 数据类型 | 处理方式 | 示例 |
|----------|----------|------|
| 密码/密钥 | 删除 | - |
| Token | Hash (SHA256) | a1b2c3... |
| 信用卡 | 掩码 | ****1234 |
| 手机号 | 部分掩码 | +86****5678 |
| 邮箱 | Hash或掩码 | a***@example.com |
| SSN | 掩码 | ***-**-1234 |

## 4.  网络安全

### 4.1 防火墙规则

**入站规则**:
`ash

# 只允许特定网段访问Collector

iptables -A INPUT -p tcp -s 10.0.0.0/8 --dport 4317 -j ACCEPT
iptables -A INPUT -p tcp -s 10.0.0.0/8 --dport 4318 -j ACCEPT
iptables -A INPUT -p tcp --dport 4317 -j DROP
iptables -A INPUT -p tcp --dport 4318 -j DROP
`

**出站规则**:
`ash

# Collector只允许访问后端存储

iptables -A OUTPUT -p tcp -d 10.0.1.0/24 --dport 4317 -j ACCEPT
iptables -A OUTPUT -p tcp --dport 4317 -j DROP
`

### 4.2 网络隔离

**推荐架构**:

- Collector部署在独立子网
- Agent与Collector同VPC
- 跨VPC使用PrivateLink/VPC Peering
- 禁止公网暴露Collector端口

## 5. � 存储安全

### 5.1 持久化队列加密

**文件存储加密**:
`yaml
exporters:
  otlp:
    sending_queue:
      enabled: true
      storage: file_storage/encrypted

extensions:
  file_storage/encrypted:
    directory: /var/lib/otelcol/queue
    timeout: 10s
    # 使用操作系统级加密 (如LUKS)
`

**检查项**:

- [ ] 磁盘加密 (LUKS/BitLocker)
- [ ] 文件权限 (chmod 700)
- [ ] 定期清理策略
- [ ] 备份加密

### 5.2 后端存储安全

| 后端 | 安全措施 |
|------|----------|
| Jaeger | TLS + 认证 |
| Tempo | mTLS + API Key |
| Prometheus | TLS + Bearer Token |
| Elasticsearch | TLS + 用户名/密码 |
| ClickHouse | TLS + 用户名/密码 |

## 6. � 访问控制

### 6.1 RBAC配置

**Kubernetes RBAC**:
`yaml
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: otel-collector
rules:

- apiGroups: [""]
  resources: ["pods", "nodes"]
  verbs: ["get", "list", "watch"]

---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: otel-collector
subjects:

- kind: ServiceAccount
  name: otel-collector
roleRef:
  kind: Role
  name: otel-collector
  apiGroup: rbac.authorization.k8s.io
`

### 6.2 最小权限原则

- [ ] Collector只读取必要的K8s资源
- [ ] 禁止访问Secrets (除非必要)
- [ ] 限制网络访问范围
- [ ] 禁止root运行Collector

## 7.  审计与监控

### 7.1 审计日志

**启用审计**:
`yaml
service:
  telemetry:
    logs:
      level: info
      output_paths: ["/var/log/otelcol/audit.log"]
    metrics:
      level: detailed
`

**审计内容**:

- [ ] 配置变更日志
- [ ] 认证失败日志
- [ ] 数据导出日志
- [ ] 错误和异常日志

### 7.2 安全监控指标

| 指标 | 告警条件 | 说明 |
|------|----------|------|
| auth_failures_total | > 10/5min | 认证失败 |
| tls_handshake_errors | > 5/5min | TLS握手错误 |
| rejected_spans_total | > 100/5min | 被拒绝的span |
| queue_size | > 80% | 队列使用率 |
| memory_usage | > 85% | 内存使用 |

## � 安全事件响应

### 发现安全事件时

1. **立即隔离**: 停止受影响Collector实例
2. **收集证据**: 导出相关日志
3. **评估影响**: 确定数据泄露范围
4. **修复漏洞**: 应用安全补丁
5. **恢复服务**: 验证后恢复
6. **复盘总结**: 更新安全策略
