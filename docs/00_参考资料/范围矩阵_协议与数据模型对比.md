> **覆盖范围**: 本项目三条内容范围（OTLP 协议、Metrics 数据模型、Logs 数据模型）

## 1. 协议维度：gRPC vs HTTP、Protobuf vs JSON

| 维度 | gRPC | HTTP/1.1 |
|------|------|----------|
| 传输层 | HTTP/2，多路复用 | HTTP/1.1 |
| 默认端口 | 4317 (TLS 4318) | 4318 (TLS 4319) |
| 流式 | 支持双向流 | 请求/响应 |
| 性能 | 高吞吐、低延迟 | 适合防火墙/简单客户端 |
| 适用场景 | 生产、服务端 | 浏览器、受限环境 |

| 维度 | Protocol Buffers | JSON |
|------|-------------------|------|
| 编码 | 二进制，紧凑 | 文本，可读 |
| 体积 | 小 | 大 |
| 解析 | 需 schema | 无需 schema |
| 适用 | 默认 OTLP 编码 | 调试、HTTP JSON 端点 |

## 2. Metrics 维度：OTLP vs Prometheus vs StatsD

| 维度 | OTLP Metrics | Prometheus | StatsD |
|------|--------------|------------|--------|
| 类型 | Counter, Gauge, Histogram, UpDownCounter, ExponentialHistogram | Counter, Gauge, Histogram, Summary | counter, gauge, timer, set |
| 聚合临时性 | Cumulative / Delta | 多为 Cumulative（拉模型） | 通常 flush 周期内聚合 |
| 模型 | ResourceMetrics → ScopeMetrics → Metric → DataPoint | 指标名 + labels → 样本 | 指标名 + 类型 + 值（可选 tags） |
| 典型用途 | 统一遥测出口 | 拉取、存储、告警 | 应用内打点、UDP 推送 |
| 与 OTLP 互操作 | 原生 | Collector receiver/exporter、Remote Write | Collector receiver/exporter |

---

## 3. Logs 维度：OTLP LogRecord vs 常见格式

| 源格式 | body 对应 | severity 对应 | 其它字段 → OTLP |
|--------|-----------|----------------|-----------------|
| OTLP LogRecord | body (AnyValue) | severity_number + severity_text | attributes, trace_id, span_id |
| JSON 日志 | message / msg | level → severity | 其余 → attributes |
| Syslog (RFC 5424) | MSG | SEVERITY | HOSTNAME→Resource; STRUCTURED-DATA→attributes |
| CloudWatch Logs | message | 若有 level | logGroup/logStream→Resource 或 attributes |
| 纯文本行 | 整行 | 可选默认 | source 等→attributes |

详见 [03_统一结构与多格式映射](../../docs/03_数据模型/03_Logs数据模型/03_统一结构与多格式映射.md)。
