## ⚡ 快速启动指南 (1天)

适合需要**立即使用**OTLP的场景。

### 上午: 环境搭建

```bash
# 1. 启动Collector (5分钟)
docker run -p 4317:4317 -p 4318:4318 \
  otel/opentelemetry-collector-contrib:0.117.0 \
  --config "$(cat config.yaml)"

# 2. 运行示例应用 (10分钟)
# 选择对应语言的SDK示例

# 3. 验证数据接收 (5分钟)
curl http://localhost:13133/health
```

### 下午: 基本使用

```markdown
📚 必看文档 (2小时)
├── 01_OTLP核心协议/01_协议概述.md
├── 07_工具生态/01_SDK指南/对应语言指南.md
└── 00_参考资料/OTLP协议速查手册.md

🔧 实践任务 (4小时)
├── 集成SDK到现有应用
├── 配置Collector
├── 查看数据导出
└── 配置基础监控
```

### 快速配置模板

```yaml
# 最小化生产配置
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
      http:
        endpoint: 0.0.0.0:4318

processors:
  batch:
    timeout: 1s

exporters:
  otlp:
    endpoint: your-backend:4317
    tls:
      insecure: true  # 生产环境请使用TLS

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
    metrics:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
    logs:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp]
```

## 🎯 学习建议

### 学习方法

1. **理论与实践结合**: 每学习一个概念，立即动手实践
2. **从简单开始**: 先跑通基本流程，再深入细节
3. **善用社区资源**: 参与讨论，提出问题
4. **记录学习笔记**: 建立自己的知识库
5. **持续跟踪更新**: 关注版本变更和新特性

### 推荐工具

- **本地开发**: Docker, Kind (本地K8s)
- **调试工具**: curl, grpcurl, jq
- **可视化**: Jaeger UI, Grafana Tempo
