# 商业观测平台与OpenTelemetry集成指南

> **涵盖**: Datadog / New Relic / Splunk / Dynatrace
> **目标**: 帮助企业选择合适的商业平台
> **适用**: 技术决策者、架构师

---

## 平台对比概览

| 平台 | OTLP支持 | 优势 | 劣势 | 定价模式 |
|------|----------|------|------|----------|
| **Datadog** | ✅ 原生 | 功能全面，生态丰富 | 价格较高 | 按主机+数据量 |
| **New Relic** | ✅ 原生 | 价格透明，按用户 | 学习曲线陡峭 | 按数据量 |
| **Splunk** | ✅ 原生 | 日志分析强 | 非常昂贵 | 按数据量 |
| **Dynatrace** | ✅ 原生 | AI分析强 | 价格较高 | 按主机 |

## 1. Datadog 集成

### 1.1 架构方案

`
Application → OTel SDK → OTLP → Datadog Agent → Datadog Cloud
                                ↓
                         Local Processing
`

### 1.2 配置步骤

**步骤1: 安装Datadog Agent**

`ash

# 安装Agent

DD_API_KEY=<YOUR_API_KEY> DD_SITE="datadoghq.com" \
bash -c ""
`

**步骤2: 启用OTLP接收**

`yaml

# /etc/datadog-agent/datadog.yaml

otlp_config:
  receiver:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
      http:
        endpoint: 0.0.0.0:4318
  metrics:
    enabled: true
  traces:
    enabled: true
  logs:
    enabled: true
`

**步骤3: 配置Application**

`yaml

# otel-collector-config.yaml

exporters:
  otlp/datadog:
    endpoint: datadog-agent:4317
    tls:
      insecure: true
    headers:
      dd-api-key: \

service:
  pipelines:
    traces:
      exporters: [otlp/datadog]
    metrics:
      exporters: [otlp/datadog]
`

### 1.3 特性映射

| OpenTelemetry | Datadog | 说明 |
|---------------|---------|------|
| trace_id | trace_id | 自动映射 |
| span_id | span_id | 自动映射 |
| service.name | service | 自动映射 |
| deployment.environment | env | 标签映射 |
| host.name | host | 自动映射 |

### 1.4 成本优化

`yaml

# 使用采样降低Datadog成本

processors:
  probabilistic_sampler:
    sampling_percentage: 10.0  # Datadog按span收费
`

## 2. New Relic 集成

### 2.1 架构方案

`
Application → OTel SDK → OTLP → New Relic Cloud (直接发送)
                ↓
         无需本地Agent
`

### 2.2 配置步骤

**步骤1: 获取License Key**

在New Relic控制台: Account Settings → API Keys → License Key

**步骤2: 配置Collector**

`yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

processors:
  batch:

exporters:
  otlp/newrelic:
    endpoint: otlp.nr-data.net:4317
    headers:
      api-key: \
    tls:
      insecure: false

service:
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp/newrelic]
    metrics:
      receivers: [otlp]
      processors: [batch]
      exporters: [otlp/newrelic]
`

### 2.3 New Relic特定功能

**无限追踪 (Infinite Tracing)**:
`yaml
exporters:
  otlp/newrelic-trace-observer:
    endpoint: \:443
    headers:
      api-key: \
`

## 3. Splunk 集成

### 3.1 架构方案

`
Application → OTel SDK → OTLP → Splunk OTel Collector → Splunk Cloud
                                ↓
                         增强数据处理
`

### 3.2 配置步骤

**步骤1: 部署Splunk OTel Collector**

`ash

# Helm安装

helm repo add splunk-otel-collector-chart <https://signalfx.github.io/splunk-otel-collector-chart>
helm install splunk-otel-collector splunk-otel-collector-chart/splunk-otel-collector \
  --set splunkObservability.realm=us0 \
  --set splunkObservability.accessToken=\ \
  --set clusterName=my-cluster
`

**步骤2: 配置Application发送OTLP**

`yaml
exporters:
  otlp/splunk:
    endpoint: splunk-otel-collector:4317
    tls:
      insecure: true
`

### 3.3 Splunk特定功能

**日志与追踪关联**:
`yaml
processors:
  resource/add_environment:
    attributes:
      - key: deployment.environment
        value: production
        action: upsert
`

## 4. Dynatrace 集成

### 4.1 架构方案

`
Application → OTel SDK → OTLP → Dynatrace ActiveGate → Dynatrace Cloud
                                ↓
                         本地处理和聚合
`

### 4.2 配置步骤

**步骤1: 部署ActiveGate**

在Dynatrace控制台: Deploy Dynatrace → Install ActiveGate

**步骤2: 启用OTLP接收**

在ActiveGate配置中启用OTLP接收端。

**步骤3: 配置Collector**

`yaml
exporters:
  otlp/dynatrace:
    endpoint: \:4317
    headers:
      Authorization: "Api-Token \"
    tls:
      insecure: false
      ca_file: /etc/ssl/certs/dynatrace.crt
`

### 4.3 Dynatrace AI功能

自动关联OpenTelemetry数据与Dynatrace的PurePath技术。

## 5. 选型建议

### 5.1 按场景推荐

| 场景 | 推荐平台 | 理由 |
|------|----------|------|
| 中小企业 | New Relic | 价格透明，按数据量 |
| 大型企业 | Datadog | 功能全面，生态丰富 |
| 日志分析为主 | Splunk | 日志分析能力强 |
| AI运维需求 | Dynatrace | AI分析能力强 |
| 成本敏感 | New Relic | 总体成本较低 |

### 5.2 迁移建议

从开源方案迁移到商业平台:

1. **并行运行**: 同时发送数据到旧系统和新平台
2. **逐步切换**: 按服务逐步切换
3. **验证数据**: 对比数据完整性
4. **培训团队**: 熟悉新平台功能
5. **完全切换**: 关闭旧系统
