### 1.1 GenAI语义约定重大增强

**状态**: Development → 更成熟
**影响**: 高

#### 新增属性

```yaml
# LLM操作属性
gen_ai.operation.name:
  - chat          # 聊天完成
  - embeddings    # 向量嵌入
  - text_completion # 文本补全
  - image_generation # 图像生成
  - audio_generation # 音频生成
  - tool_calling  # 工具调用

# 模型属性
gen_ai.model.id:              # 模型唯一标识
gen_ai.model.provider:        # 模型提供商
  - openai
  - anthropic
  - google
  - azure
  - aws.bedrock
  - cohere
  - meta
  - mistral
  # ... 等

# Token使用
gen_ai.token.count:           # 总Token数
gen_ai.token.input:           # 输入Token
gen_ai.token.output:          # 输出Token

# 成本属性 (新增)
gen_ai.cost.currency:         # 货币单位 (USD, CNY等)
gen_ai.cost.amount:           # 成本金额
gen_ai.cost.model_price_per_token: # 每Token价格
```

#### 使用示例

```python
# Python SDK示例
from opentelemetry import trace
from opentelemetry.semconv.ai import GEN_AI_ATTRIBUTES

tracer = trace.get_tracer(__name__)

with tracer.start_as_current_span("chat_completion") as span:
    # 设置GenAI属性
    span.set_attribute("gen_ai.system", "openai")
    span.set_attribute("gen_ai.operation.name", "chat")
    span.set_attribute("gen_ai.model.id", "gpt-4")
    span.set_attribute("gen_ai.model.provider", "openai")
    span.set_attribute("gen_ai.token.input", 150)
    span.set_attribute("gen_ai.token.output", 200)
    span.set_attribute("gen_ai.token.count", 350)
    span.set_attribute("gen_ai.cost.currency", "USD")
    span.set_attribute("gen_ai.cost.amount", 0.012)
```

### 1.2 数据库语义约定更新

**状态**: Stable
**影响**: 中

#### 新增数据库类型


### 1.3 HTTP语义约定微调

**状态**: Stable
**影响**: 低

#### 属性重命名 (废弃旧属性)

| 旧属性 (v1.38-) | 新属性 (v1.39+) | 状态 |
|:----------------|:----------------|:----:|
| `http.request.body.size` | `http.request.body.size` | 无变化 |
| `http.response.body.size` | `http.response.body.size` | 无变化 |

#### 新增属性

```yaml
# HTTP内容协商
http.request.header.accept-encoding:
http.request.header.accept-language:
http.response.header.content-encoding:

# HTTP缓存
http.request.cache.hit:       # 缓存命中
cache.key:                    # 缓存键
cache.ttl:                    # 缓存TTL
```

## 二、稳定性变更

### 2.1 升级到Stable的约定

| 约定领域 | v1.38状态 | v1.39状态 | 说明 |
|:---------|:---------:|:---------:|:-----|
| HTTP Client/Server | Stable | Stable | 无变化 |
| Database | Stable | Stable | 无变化 |
| Messaging (基础) | Experimental | **Stable** | 正式稳定 |
| RPC (gRPC) | Stable | Stable | 无变化 |
| GenAI | Development | Development | 持续开发 |
| System | Experimental | **Stable** | 正式稳定 |
| Process | Stable | Stable | 无变化 |

### 2.2 废弃的属性

| 属性 | 替代属性 | 废弃版本 | 移除计划 |
|:-----|:---------|:--------:|:--------:|
| `http.flavor` | `http.protocol.version` | v1.39 | v2.0 |
| `net.host.name` | `server.address` | v1.39 | v2.0 |
| `net.host.port` | `server.port` | v1.39 | v2.0 |
| `net.peer.name` | `server.address` | v1.39 | v2.0 |
| `net.peer.port` | `server.port` | v1.39 | v2.0 |

## 三、迁移指南

### 3.1 从v1.38迁移到v1.39

#### 步骤1: 更新依赖

```bash
# Go SDK
go get go.opentelemetry.io/otel/semconv@v1.39.0

# Python SDK
pip install opentelemetry-semantic-conventions==0.50b0

# Java SDK
<dependency>
    <groupId>io.opentelemetry</groupId>
    <artifactId>opentelemetry-semconv</artifactId>
    <version>1.39.0-alpha</version>
</dependency>

# Node.js SDK
npm install @opentelemetry/semantic-conventions@1.29.0
```

#### 步骤2: 替换废弃属性

```python
# 旧代码 (v1.38)
span.set_attribute("net.host.name", "example.com")
span.set_attribute("net.host.port", 8080)

# 新代码 (v1.39)
span.set_attribute("server.address", "example.com")
span.set_attribute("server.port", 8080)
```

#### 步骤3: 采用新属性

```python
# 新增GenAI监控
span.set_attribute("gen_ai.cost.currency", "USD")
span.set_attribute("gen_ai.cost.amount", 0.05)
```
