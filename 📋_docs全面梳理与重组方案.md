# docs文件夹全面梳理与重组方案

> **目标**: 建立统一的命名规范，优化目录结构，消除重复，提升导航体验
> **范围**: docs目录全部267篇文档
> **原则**: 主题相关性、命名一致性、结构扁平化

---

## 1. 现状分析

### 1.1 结构问题

```
问题清单:
├── 新旧目录并存
│   ├── 旧: 01_OTLP核心协议/ (6篇)
│   ├── 新: 02_核心协议层/ (0篇) ← 空目录
│   └── 需要统一合并
│
├── 命名不规范
│   ├── 前缀混乱: 参考_、指南_、快速_
│   ├── 缺少编号: 部分文档无NN_前缀
│   └── 需要统一: NN_类别_主题.md
│
├── 内容分散
│   ├── 同一主题分散多处
│   ├── 重复内容存在
│   └── 需要集中归类
│
└── 层级过深
    ├── 部分路径超过3层
    └── 需要扁平化
```

### 1.2 命名问题统计

| 问题类型 | 数量 | 示例 |
|----------|------|------|
| 无编号前缀 | ~50篇 | `参考_TLA+模型检验...md` |
| 前缀混乱 | ~80篇 | `指南_AI_...md`, `快速_...md` |
| 中英文混合 | ~30篇 | `AI_ML_可观测性...md` |
| 特殊字符 | ~20篇 | `✅_`, `📊_`, `📝_` |

## 2. 统一命名规范

### 2.1 文档命名格式

```
格式: NN_类别_主题名称.md

示例:
├── 01_协议_OTLP协议概述.md
├── 11_模型_Traces数据模型.md
├── 21_采样_采样策略概述.md
├── 31_部署_Kubernetes部署指南.md
└── 41_案例_电商系统可观测性实战.md

规则:
1. NN: 2位数字编号 (01-99)
2. 类别: 2-4字中文类别名
3. 主题: 清晰简洁的主题描述
4. 扩展名: .md
```

### 2.2 类别词汇表

| 类别 | 用途 | 适用层级 |
|------|------|----------|
| 协议 | OTLP协议规范 | Layer 2 |
| 模型 | 数据模型 | Layer 2 |
| 约定 | 语义约定 | Layer 2 |
| 理论 | 理论基础 | Layer 1 |
| 实现 | 实现技术 | Layer 3 |
| 采样 | 采样策略 | Layer 3 |
| 部署 | 部署运维 | Layer 4 |
| 故障 | 故障排查 | Layer 4 |
| 安全 | 安全合规 | Layer 4 |
| 案例 | 实战案例 | Layer 5 |
| 分析 | 数据分析 | Layer 5 |
| 指南 | 工具指南 | Layer 6 |
| 参考 | 参考资料 | Layer 6 |
| 项目 | 项目管理 | Layer 7 |

### 2.3 目录命名格式

```
格式: NN_中文主题/

示例:
├── 01_基础理论层/
├── 02_核心协议层/
│   ├── 11_Traces数据模型/
│   └── 12_Metrics数据模型/
└── 03_核心实现层/

规则:
1. NN: 2位数字编号
2. 主题: 中文描述
3. 子目录: 使用NN_前缀继续编号
```
